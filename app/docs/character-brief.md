# Character art brief

The three characters shipped in V1 (`Berry`, `Drip`, `Sprout`) are **procedural
placeholders** generated by `scripts/generate_assets.py`. They're clean and
readable, but real commissioned pixel art will make the app feel premium. This
brief is what to hand a pixel artist — the format they must deliver in never
changes, so new art drops straight in.

## Deliverable format (per character)

A single **sprite-sheet PNG** (`atlas.png`) + a `manifest.json`, placed in
`public/characters/<id>/`.

- **Frame size:** 160 × 192 px, transparent background.
- **Layout:** columns = animation frames, rows = states, in this order:
  - row 0 — `idle` (2 frames): gentle breathing/bob loop
  - row 1 — `walk` (2 frames): walk cycle (the app slides the character across
    the screen; these frames sell the stepping)
  - row 2 — `drink` (2 frames): raise a tumbler and sip; frame 2 is the happy,
    eyes-closed sip
- **Anchor:** feet centered horizontally, standing near the bottom (`anchorY`
  ≈ 176). Keep a few px of headroom at the top for the speech bubble.
- **Style:** true pixel art on a consistent grid; limited shared palette across
  the cast so they read as one family. The buddy should hold/sip from a
  **Strawvarie tumbler** in the drink frames.

## The cast

| id | character | note |
| --- | --- | --- |
| `berry` | Strawberry mascot | the Strawvarie mascot; make this the charmer |
| `drip` | Water droplet | classic, friendly, simple |
| `sprout` | Leafy sprout | ties to "Sustainable at Heart" |

## Future states (not needed for V1)

If the animation set expands later, the engine already supports adding rows:
`greeting`, `celebrate` (goal moments), `concerned` (after ignored reminders),
`snoozed`. Just add them to `states` in the manifest with their row/frame counts
— no code change.
